栈

what is 栈 ?

正常说法：一系列对象组成的一个集合，这些对象的插入和删除操作遵循后进先出(LIFO)的原则。

形象说法：类似枪械的弹匣，越是后压进去的子弹越先发射

栈的抽象数据类型

栈是一种支持下面操作的抽象数据类型(ADT)，用S表示这一ADT的实例：

  • S.push(e): 将元素e添加到栈S的栈定，俗称压栈

  • S.pop(e): 从栈S中移除并返回栈顶的元素，若此时为空栈，则操作出错

以上两种操作是栈的典型操作

  • S.top(): 在不移除栈顶元素的情况，返回一个栈S的栈顶元素，若空栈，则操作出错

  • S.is_empty(): 判断栈S是否包含元素， 若不包含任何元素，返回布尔值 True

  • len(S): 返回栈S中元素的数量

链表

什么是链表？

链表是物理存储单元上非连续、非顺序的存储结构，数据元素的逻辑顺序是通过链表的指针地址实现，每个元素包含两部分一是数据域一是指针域，根据指针的指向，链表能形成不同的结构，单链表、双向链表、循环链表。。。

单向链表

单向链表是由多个节点的集合共同构成一个线性序列。每个节点存储一个对象的引用，这个引用指向链表的下一个节点，若没有下一个节点了，则为空。

队列

啥是队列 ？

正经说法：有一系列对象组成的集合，这些对象的插入和删除遵循先进先出(FIFO)的原则，也就是说元素入队列随时都可以，但是出队列必须出的是队列最前面的元素。

形象说法：早上排队买早点，讲究先来后到，先排队的自然先买，不存在插队等没素质的行为，O(∩_∩)O哈哈~

队列的抽象数据类型

队列的抽象数据类型定义了一个包含一系列对象的集合，其中元素的访问和删除被限制在队列的第一个元素，而元素的插入被限制在了队列的尾部。

对于队列Q，队列的抽象数据类型(ADT)支持如下两个基本方法：

  • Q.enqueue(e): 元素e的入队列操作
  • Q.dequeue(): 从队列Q中移除并返回第一个元素，若队列为空，操作出错

队列的抽象数据类型(ADT)还包括如下方法：

  • Q.first(): 返回队列的第一个元素，且不删除该元素。若队列为空，操作出错
  • Q.is_empty(): 如果队列不包含任何元素，返回布尔值 True
  • len(Q): 返回队列Q的元素数量，python中可以用len()实现

双端队列

what is the 双端队列？

队列的两端都支持插入和删除操作，拽下洋文(double-ended queue或者deque)

双端队列的抽象数据类型

定义双端队列的抽象数据类型D，该ADT支持如下方法：

  • D.add_first(e): 向双端队列的前面添加一个元素e

  • D.add_last(e): 向双端队列的后面添加一个元素e

  • D.delete_first(): 从双端队列中移除并返回第一个元素，若双端队列为空，则触发错误

  • D.delete_last(): 从双端队列中移除并返回最后一个元素，若双端队列为空，则触发错误

  • D.first(): 返回但不移除双端队列的第一个元素，如果双端队列为空，则触发错误

  • D.last(): 返回但不移出双端队列的最后一个元素，如果双端队列为空，则触发错误

  • D.is_empty(): 如果双端队列为空，返回True

  • len(D): 返回当前双端队列的元素个数，python中用len()实现

小结一下：

  • 栈、队列和链表都属于线性结构

  • 大致分为两类

    • 基于数组

      • 访问元素，由于提供索引访问，所以基于数组的序列访问元素的操作，时间复杂度均为O(1)
      • 插入和删除元素，在基于数组的序列尾端进行插入删除操作，只有O(1)，在其余位置上的插入删除操作，则可以看作O(n)

    • 基于链表

      • 访问元素，通常都需要遍历链表，最好的时间复杂度是O(1)，最差则需要O(n)
      • 插入和删除元素，基于链表的序列在进行插入删除操作，时间复杂度均为O(1)，上述实现链表的中间元素的插入和中间元素的删除，并没有O(1)，这么快，是因为没有提供插入位置的两端的节点，只是通过计数遍历去实现的插入或删除。

  • 书上说的，没有十全十美的数据结构，基于数组也好基于链表也罢，都各有长短。根据场景选取比较适合的数据结构，是个需要长期修炼的课题。
