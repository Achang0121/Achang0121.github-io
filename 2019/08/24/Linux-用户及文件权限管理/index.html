创建用户

在Linux系统里，root账户拥有整个系统至高无上的权利。拥有root权限可以对系统的任何文件执行增、删、改、查的操作。

su，su- 与 sudo

  1. su <user> 可以切换到user，执行的时候需要输入目标用户的密码

  2. sudo <cmd> 可以以特权级别运行cmd命令，需要当前用户属于sudo组，且需要输入当前用户的密码

  3. su - <user> 命令也是切换用户，同时环境变量也会跟着改变成目标用户的环境变量

用户组

顾名思义，用户组就是一组用户的集合，它们共享一些资源和权限，同时也拥有私有资源和权限。

  • groups <用户名> 可查看该用户的用户组。返回的结果是 –> 用户名:用户组名 (若该用户创建的时候未制定用户组的话，默认会自动创建一个与用户名相同的用户组)

  • cat /etc/group 这种方式也是能够看到用户组的信息，在数据较多的情况下，可以使用命令过滤掉不想看的结果(cat /etc/group | grep -E “user” )

  • /etc/group 文件格式说明

    格式 group_name:password:GID:user_list

    由于Linux中密码不可见所以显示的都是x，当用户的GID与用户组的GID相同时，那么最后的一个字段user_list就是空的

  • 将其他用户加入sudo用户组

Linux文件权限

在Linux中的每一个文件或目录都包含有访问权限，这些权限决定了谁能访问和如何访问这些文件和目录。

修改文件权限的两种方式

  • 方式一 : 二进制数字表示

每个文件的三组权限(拥有者，所属用户组，其他用户，顺序是一定的)对应一个’rwx’(读写执行)

  • 方式二 : 加减赋值操作

g、o 、u分别表示group、others和user

+ 和- 分辨表示增加和去掉相应的权限

  • adduser 和 useradd 的区别是什么？

    1. useradd 只创建用户，创建完了用 passwd lilei 去设置新用户的密码。
    2. adduser 会创建用户，创建目录，创建密码（提示你设置），做这一系列的操作。
    3. 其实 useradd、userdel 这类操作更像是一种命令，执行完了就返回。而 adduser 更像是一种程序，需要你输入、确定等一系列操作。
