237. 删除链表中的节点 <<<<=== 题目传送门

  • 题目描述

    请编写一个函数，使其可以删除某个链表中给定的（非末尾）节点，你将只被给定要求被删除的节点。

    现有一个链表 – head = [4,5,1,9]，它可以表示为:

    示例 1:

    输入: head = [4,5,1,9], node = 5
    输出: [4,1,9]
    解释: 给定你链表中值为 5 的第二个节点，那么在调用了你的函数之后，该链表应变为 4 -> 1 -> 9.
    示例 2:

    输入: head = [4,5,1,9], node = 1
    输出: [4,5,9]
    解释: 给定你链表中值为 1 的第三个节点，那么在调用了你的函数之后，该链表应变为 4 -> 5 -> 9.

    说明:

    链表至少包含两个节点。
    链表中所有节点的值都是唯一的。
    给定的节点为非末尾节点并且一定是链表中的一个有效节点。
    不要从你的函数中返回任何结果。

  • 题解

    初步想法：删除这个指定的node，就是用它前一个node的next指向它后一个node，将这个node跳过去即可。从head去依次查找待删除的node，找到后按之前的想法操作就ok了。

    但是看了网站给的代码，deleteNode函数只给了待删除的node。并不能获取到node的前一个节点，只能通过node.next获取其之后的节点，所以考虑去替换node的val和next。

21. 合并两个有序链表 <<<<=== 题目传送门

  • 题目描述

    将两个有序链表合并为一个新的有序链表并返回。新链表是通过拼接给定的两个链表的所有节点组成的。

    示例：

    输入：1->2->4, 1->3->4

    输出：1->1->2->3->4->4

  • 个人题解

    因为之前写过合并两个有序的列表，所以合并两个有序链表其实差不太多。区别就在列表和链表的遍历方式不一样，其余部分都差不多。都是通过比较两个链表的元素，取较小值，放入新的链表。

  • 大佬题解

    递归的解法是我没想到的，可能迭代用得比较多吧，也更容易想到。。。

2. 两数相加 <<<<=== 题目传送门

  • 题目描述
    给出两个 非空 的链表用来表示两个非负的整数。其中，它们各自的位数是按照 逆序 的方式存储的，并且它们的每个节点只能存储 一位 数字。

    如果，我们将这两个数相加起来，则会返回一个新的链表来表示它们的和。

    您可以假设除了数字 0 之外，这两个数都不会以 0 开头。

    示例：

    输入：(2 -> 4 -> 3) + (5 -> 6 -> 4)

    输出：7 -> 0 -> 8

    原因：342 + 465 = 807

  • 个人题解

    看它的示例，个人第一感觉是，先将两个链表转换成数值，再进行相加，最后用和的值构建结果链表。

    结果表明，这思路也能够完成题目的需求。这个网站的测试用例应该没有那种，100位数或者更多位数的。如果那样的话，估计就得歇菜了。

    再读题后，发现它逆序是有道理的，小学数学加法竖式计算，第一个要求就是相同的数位对齐。所以l1和l2其实已经做了这一个操作，尽管是左右相反的，但并不影响。模拟加法进位的方式，是不是可以试一试。

    试了半天，感觉这如果两个链表不一样长的时候，短链表取值结束后，后面的处理比较的麻烦。所以打算补0，把两个链表弄一样长，这样我就不管你短还是长，以长的为准。

    运行完了，这么写好像更慢了，比较起来还不如上面的写法。

  • 大佬题解

    都是用的模拟加法进位来写的，大同小异。个人还是觉得第一种，“愣头青”式的解法，不需要考虑进位不进位，是否长短不一样，更直白点。而且网站的运行结果还好点。
